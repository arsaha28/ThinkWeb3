ISO20022

ISO 20022 is a global standard for electronic data interchange between financial institutions. It provides a common language and syntax for financial messages, enabling communication across borders and between different systems and applications.

SEPA Transfer Steps

Customer-to-Bank Transfer

Here's an example of the steps involved in a customer-to-bank transfer using the SEPA Credit Transfer (SCT) scheme:

  1. The customer initiates the payment instruction through their bank's online banking portal or other electronic channels.
  2. The customer's bank validates the payment instruction and sends it to the clearing and settlement mechanism (CSM) responsible for processing SCT payments.
  3. The CSM processes the payment instruction and sends it to the receiving bank.
  4. The receiving bank credits the recipient's account with the payment amount.
  5. The receiving bank sends a confirmation message (such as a PAIN.002 message) to the sending bank, indicating that the payment has been received and credited to the recipient's account.
  6. The sending bank sends a confirmation message to the customer, indicating that the payment has been successfully processed and credited to the recipient's account.


PAIN Message Definition

A PAIN message is a type of ISO 20022 message used for initiating and reporting payment transactions. PAIN stands for "Payments Initiation". PAIN messages are used in the Single Euro Payments Area (SEPA) and other regions to enable cross-border payments in a standardized and secure way.

There are two commonly used PAIN message types:

  • PAIN.001: This message type is used for initiating credit transfers and direct debits.
  • PAIN.002: This message type is used for reporting on the status of credit transfers and direct debits.

PACS Message Definition

PACS stands for "Payments Clearing and Settlement". It is a type of ISO 20022 message used for clearing and settlement of payment transactions. PACS messages are used in the Eurozone and other regions.

There are several PACS message types, including:

  • PACS.002: This message type is used for reporting on the status of payment instructions.
  • PACS.004: This message type is used for indicating that a payment has been accepted or rejected by the receiving bank.
  • PACS.008: This message type is used for initiating FI to FI Customer Credit Transfer.
  • PACS.009: This message type is used for initiating FI to FI Institution Credit Transfer.

CAMT Message Definition

CAMT stands for "Cash Management". It is a type of ISO 20022 message used for cash management purposes, such as account reporting and payment reconciliation.

There are several CAMT message types, including:

  • CAMT.052: This message type is used for Bank To Customer Account Report.
  • CAMT.053: This message type is used for Bank to Customer Statement.
  • CAMT.054: This message type is used for Bank to Customer Debit Credit Notification.
  • CAMT.055: This message type is used for Customer Payment Cancellation Request.
  • CAMT.029: This message type is used for reconciliation of payment-related information.
